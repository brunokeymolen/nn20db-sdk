# nn20db GeoNames Demo

Nearest-neighbour city search using the [nn20db](../../../../README.md) vector database and the [GeoNames](https://www.geonames.org/) `cities15000` dataset.

## What it does

1. Downloads `cities15000.zip` from GeoNames on first run and caches it in `data/`.
2. Loads the first 10,000 cities from the dataset.
3. Converts each city's latitude/longitude into a **3D unit-sphere vector**.
4. Inserts all vectors into an nn20db HNSW index stored in `data/geonames_10k_db/`.
5. Runs four example geo queries (Brussels, Paris, New York, Tokyo) and prints the top 10 nearest cities for each, with exact haversine distances.

## How to run

Build the Python API shared library first (once):

```bash
make -C api/python
```

Then, from the repository root:

```bash
python demos/geo/linux/python/demo_geonames_10k.py
```

On subsequent runs the cached dataset and index are reused automatically.
To force a clean rebuild of the index:

```bash
python demos/geo/linux/python/demo_geonames_10k.py --rebuild
```

Additional options:

```
--max-cities N   Index only the first N cities (default: 10000)
--top-k K        Return top K nearest neighbours (default: 10)
--rebuild        Delete and recreate the index
```

## The vector representation

Each city at coordinates (lat°, lon°) is mapped to a point on the unit sphere in ℝ³:

```
x = cos(lat) · cos(lon)
y = cos(lat) · sin(lon)
z = sin(lat)
```

This is the standard Cartesian embedding of spherical coordinates. The resulting 3D vectors have unit length (`|v| = 1`).

## Why 3D unit vectors?

Nearest-neighbour search on the unit sphere is equivalent under both L2 and cosine similarity — for unit vectors `|a| = |b| = 1`, the relationship

```
||a - b||² = 2 - 2·cos(θ)
```

means that minimising L2 distance is identical to maximising cosine similarity, which is identical to minimising the great-circle angle θ.

The demo uses the `cosine` metric. Returned vector distances can be converted to approximate angular distances via `θ = acos(1 - d)`, and then to kilometres via `km = 6371 · θ`. However, the demo uses the exact haversine formula on the original lat/lon coordinates for display.
